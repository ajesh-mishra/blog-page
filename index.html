Python's special __name__ variable

Ajesh Mishra
 26th July, 2020   |    5 mins read

Have you ever come across the below Python code and wondered what it was. Let's try to demistify it! 

if __name__ == '__main__':

What is __name__?

Python has many special variables which starts and ends with two underscores '_' and they are used internally to keep track of things. __name__ is one of the special variables and Python uses it to store the name of the module.

Let's inspect what's in the special variable __name__: test.py

print(f'__name__ is {__name__}')

Output:

$ python3 test.py
__name__ is __main__

So, __name__ has the string __main__.

The conclusion is: Python has give a special name to the module which is __main__.

That's cool ! How about we import the module test.py into our Python REPL and check the output.

>>> import test.py
__name__ is test

So, this means __name__ contains the string test and the module name is test This is because in Python every .py file is a module and in our example test is the module name.

There are various other special variables that Python has. Check them out!

Use Case

Since Python makes use of __name__ to store the module name, developers exploite this to test there functions in the module by directly calling the functions benith the conditional if __name__ == __main__.

Consider the following code: test.py

def show():
  print('Hello World!')

if __name__ == '__main__':
  show()

Output:

$ python3 test.py
Hello World!

But, if we were to import the module test into another python file then the function won't get executed.
